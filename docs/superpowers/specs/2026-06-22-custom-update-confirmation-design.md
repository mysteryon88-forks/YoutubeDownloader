# Custom-dialog подтверждения обновления

## Цель

Заменить стандартный Windows `MessageBoxW`, предлагающий установить найденное обновление приложения, на тёмный custom-dialog в едином стиле YoutubeDownloader.

## Область изменения

- Сценарий «доступна новая версия» использует новый тип custom-dialog `Confirmation`.
- Существующие информационные и ошибочные runtime-окна продолжают использовать `ShowInfoDialog` и `ShowErrorDialog`.
- Системные `MessageBoxW` сохраняются только для четырёх фатальных ошибок до полноценного запуска UI: сбой GDI+, сбой регистрации класса кнопок, сбой регистрации класса главного окна и сбой создания главного окна.

Эти ранние ошибки нельзя надёжно отобразить custom-dialog, потому что он зависит от тех же графических и оконных компонентов.

## Содержимое

- Заголовок окна: «Обновление доступно».
- Подзаголовок внутри панели: «Доступно обновление приложения».
- Основной текст показывает найденную версию, текущую версию, предложение установить обновление и предупреждение о закрытии и повторном запуске приложения.
- Текст отображается в существующей прокручиваемой области сообщения.

## Кнопки и результат

- Справа снизу расположены две кнопки с существующими размерами, промежутком и отступами панели.
- «Позже» — вторичная кнопка, расположенная слева.
- «Установить» — акцентная кнопка, расположенная справа.
- «Установить» закрывает dialog и возвращает `true`.
- «Позже», Escape и закрытие крестиком возвращают `false`.
- Кнопки «Скопировать» и `OK` для типа `Confirmation` не создаются.

## Архитектура

- В `DialogType` добавляется `Confirmation`.
- `DialogState` получает указатель на булев результат подтверждения.
- Общие `ShowModal`, `DrawMessageDialog`, `LayoutMessageDialog`, `CreateScrollText`, custom-кнопки и буферная отрисовка переиспользуются без создания отдельного класса окна.
- Внутренний helper `ShowConfirmationDialog` создаёт состояние, запускает modal loop и возвращает выбор пользователя.
- `OfferAppUpdate` вызывает helper вместо `MessageBoxW`; дальнейшая загрузка запускается только при результате `true`.

## Визуальный стиль

Диалог использует существующие фон, панель, типографику, скругления, muted-текст, hover/pressed состояния и тёмный title bar. Новых цветов, шрифтов или иконок не добавляется.

## Проверка

- Release-сборка и полный CTest.
- Визуальная проверка окна при доступной новой версии.
- Проверка текста обеих версий и предупреждения о перезапуске.
- «Позже», Escape и крестик не запускают загрузку.
- «Установить» открывает custom progress-окно обновления.
- Поиск по исходникам подтверждает отсутствие стандартных `MessageBoxW` в runtime-сценариях; остаются только четыре документированных startup fallback-вызова.
